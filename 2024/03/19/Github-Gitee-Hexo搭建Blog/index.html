下载Node.js

官网Node.js (nodejs.org)，我搜到的都说下LTS的长稳版，current下了你就是小白鼠嘻嘻

win+r然后cmd进去输一下node -v，出来版本号就说明装过了，报错就没装，有问题复制一下报错搜一下基本都能解决

我一般都不下到C盘，都是装在D盘，下之前新建个文件夹写下nodejs这样自己知道是啥

配置环境

在安装的文件夹【D:\Develop\nodejs】下创建两个文件夹【node_global】及【node_cache】

创建完两个空文件夹之后，打开cmd命令窗口，输入

把后面下依赖包什么的全局和缓存地址改到D盘去，不然默认就在C盘了

直接在开始菜单旁边的搜索里输环境或者系统什么的，就会出来【编辑系统环境变量】，然后就【高级】-【环境变量】

在【系统变量】下新建【NODE_PATH】，输入【D:\Develop\nodejs\node_global\node_modules】

将【用户变量】下的【Path】修改为【D:\Develop\nodejs\node_global】，跟前面对应的。改完就可以了

安装Hexo

去d或者e盘建个新文件夹进去cmd，有的人可能没管理权限容易报错，建议以管理员身份运行cmd

npm在你装node的时候就给你装好了，这是在用npm初始化一个hexo-cli的脚手架，类比vue-cli一样，就是一个笼统的框架

输入hexo 按回车，出现help 就说明安装成功，你要觉得博客装的不好，比如主题删的都在飘红（没错就是我）想重弄，直接删了整个文件夹从这一步重新开始好了，反正很快（前提是你不心疼你的写了很多的博客）

推荐再装个vscode，很好下，下一步下一步就完事了

初始化项目

blog是博客名字可以改，你取个myblog啥的也行，这还是本地的

就是清一下已存在的hexo网站文件，generate生产，save保存，现在都是本地的静态页面，出现【http://localhost:4000】就ok了

改主题（可选）

然后可以去改个主题折腾一下，官网https://hexo.io/themes/随便预览一下，喜欢哪个就大概看一下，最好是说直接放在根目录的themes文件夹里就行的，整个文件导的就算了，会改的崩溃然后重来的（比如我）

如果你要改主题用模板，一般都是下载下来放到themes里，比如我下的flexblock，文件夹下下来就是themes\flexblock，然后flexblock里一堆主题有关的文件

改仓库配置

如果你用了别人的主题，就要在【blog】的博客文件下跟source、themes、node_modules同一级的【_config.yml】配置文件最后一点

把theme：lanscape改为theme：flexblock（你下的主题名字，文件夹也得叫这个名字，不换主题就不用管，只填地址就好了）

然后配置你的博客github或者gitee仓库地址，就在最后的Deployment里，本来很空，只有个type，你得把地址repo加进去，上面是github和gitee的，也能推成功，但是gitee上的没太部署好，主题是错的，github上的很正常，我用两个仓库一套公钥想一次性推但是没成功QAQ

开启Pages

以上，代码传上去了，没域名能看到博客的效果

所以需要开启Gitee的Pages功能，不然看不到页面的，你要访问的博客页面就得开Pages，这个是免费的，但是会要先申请一下权限我记得

进入码云的仓库，克隆/下载按钮上面有一排，代码、issue什么的，最后一个是管理，点击倒数第二个服务，出来的第一个就是Gitee Pages，点击进行开启，分支默认是master，也不用点别的，直接点击更新就行，然后部署分支上面就会出来一行字：

已开启Gitee Pages服务，网站地址：Hexo (gitee.io)

这就是你的域名，点进去就能看到跟本地博客一样的布局

不过Gitee的缺点就是你每次在本地写完hexo g然后hexo d，只是推送到仓库了，部署还得来Pages页面点更新才有效果，不然更新的页面是不变的
